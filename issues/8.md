# Issue 8: Tab Consistency - Generated Images Persistence

## Status: Open

## Problem Statement:
When generating images on the first tab (Image Generation), then navigating to other tabs and returning, the generated images are no longer visible. This breaks the user experience and makes it difficult to work with generated content across different workflow stages.

## Implementation Steps:
1. **Analyze Current State Management**: 
   - Investigate how images are stored/displayed in the New Canvas view
   - Check if images are cleared on tab change or if it's a display issue

2. **Implement Persistent Storage**:
   - Create a session-based image storage system
   - Ensure generated images persist across tab switches
   - Update display logic to reload images when returning to tab

3. **Cross-Tab Consistency**:
   - Ensure generated images are accessible from other tabs (e.g., for 3D generation)
   - Maintain selection states across tab switches
   - Synchronize image grids between New Canvas and View All tabs

4. **Memory Management**:
   - Implement smart caching to prevent memory issues with many images
   - Consider thumbnail generation for better performance
   - Add cleanup for old session data

## Files to Modify:
- `src/core/app_redesigned.py` - Tab change handlers and image state management
- `src/core/app_ui_methods.py` - New Canvas view creation and image grid updates
- `src/ui/widgets.py` - ImageGridWidget persistence logic
- `src/core/file_monitor.py` - Session image tracking
