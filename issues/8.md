## 🔍 Root Cause Analysis

### **Working Reference: Texture Generation Page**
**What Works There**:
```python
# Texture parameters use direct widget creation
quality_layout.addWidget(res_label)
quality_layout.addWidget(self.texture_res_combo)
# No complex size policies
# No scroll areas
# Simple QVBoxLayout with addStretch() at end
```

### **Failing Code: Dynamic Parameters**
**What's Different**:
```python
# Uses scroll area container
scroll_area = QScrollArea()
# Uses dynamic widget creation with complex layouts
param_widget = self._create_parameter_widget(name, value)
# Multiple container layers: scroll_area → container → widget
```

### **Key Differences Identified**:
1. **Container Layers**: Texture = direct addition, Dynamic = scroll area wrapper
2. **Widget Creation**: Texture = direct QComboBox, Dynamic = custom widget wrapper  
3. **Layout Complexity**: Texture = simple addWidget(), Dynamic = complex parameter widget
4. **Size Policy Inheritance**: Multiple container layers override widget policies

---

## 🎯 Recommended Next Approach

### **Strategy 1: Remove Scroll Area (Simplify)**
- Try removing scroll area completely for image parameters
- Use direct widget addition like texture parameters
- Add `layout.addStretch()` at end

### **Strategy 2: Fix Container Chain**
- Investigate scroll area size policies
- Check if `scroll_area.widget()` inherits wrong policies
- Set explicit policies on each container level

### **Strategy 3: Direct Widget Creation (Bypass Custom Method)**
- Don't use `_create_parameter_widget()` 
- Create widgets directly like texture parameters:
```python
param_label = QLabel("Parameter:")
param_label.setObjectName("section_title")
param_widget = QComboBox()
section_layout.addWidget(param_label)
section_layout.addWidget(param_widget)
```

### **Strategy 4: Container Policy Chain Investigation**
- Debug the exact size policy inheritance chain:
  - `QScrollArea` → `container_widget` → `layout` → `parameter_widget`
- Set explicit policies at each level
- Use `widget.sizeHint()` to debug preferred sizes
