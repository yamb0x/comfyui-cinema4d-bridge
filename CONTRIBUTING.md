# Contributing to ComfyUI to Cinema4D Bridge

Thank you for your interest in contributing! We welcome contributions from the community.

## 🚀 Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Follow the development setup in [docs/DEVELOPMENT_GUIDE.md](docs/DEVELOPMENT_GUIDE.md)

## 🎯 How to Contribute

### 🐛 Reporting Bugs
- Use the GitHub Issues tab
- Include system information (OS, Python version, Cinema4D version)
- Provide steps to reproduce
- Include relevant error messages from logs

### 💡 Suggesting Features
- Check existing issues first
- Describe the use case
- Explain how it benefits users
- Consider implementation complexity

### 🔧 Code Contributions

1. **Follow Code Style**
   - Use type hints
   - Follow PEP 8
   - Keep functions focused and small
   - Document complex logic

2. **Logging Guidelines** (Updated June 19, 2025)
   - NO emoji decorations in logs
   - Use `debug` for detailed information
   - Use `info` sparingly for user-facing events
   - Always show warnings and errors

3. **Testing**
   - Test your changes thoroughly
   - Ensure existing features still work
   - Add test cases for new features

4. **Documentation**
   - Update relevant docs
   - Add docstrings to new functions
   - Include usage examples

## 📝 Pull Request Process

1. **Before Submitting**
   - Ensure code follows project standards
   - Update documentation if needed
   - Test on Windows (primary platform)
   - Run the application and verify changes

2. **PR Description**
   - Clearly describe the changes
   - Reference any related issues
   - Include screenshots for UI changes
   - List any breaking changes

3. **Review Process**
   - Address reviewer feedback
   - Keep PR focused on one feature/fix
   - Be patient and respectful
